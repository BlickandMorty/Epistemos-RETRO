## Identity

Epistemos Retro Edition is a **1:1 port** of the macOS Opulent Edition to Windows. Same features, same logic, same pipeline, same graph, same vault sync. Different tech stack.

| macOS (Source of Truth) | Windows Retro (Port) |
|------------------------|---------------------|
| SwiftUI + AppKit | React 19 + Vite + Tailwind 4 |
| SwiftData | rusqlite (hand-written SQL) |
| Metal + custom force sim | Bevy + wgpu + Rapier3D |
| Apple Intelligence | Foundry Local + Ollama |
| Accelerate SIMD | ort crate (ONNX Runtime + DirectML) |
| Swift @Observable | Zustand stores |
| Tauri 2.x IPC | Tauri invoke() + events |

## Architecture

```
User → React 19 UI → Tauri invoke() → Rust Commands → Services
                   ← Tauri events  ←                ← (streaming)

Rust Services:
  ├── crates/storage  — rusqlite DB, types, IDs
  ├── crates/engine   — LLM providers, pipeline, triage, SOAR, signals
  ├── crates/graph    — GraphStore, builder, entity extractor, search
  ├── crates/sync     — Vault export/import, block parser, file watcher
  ├── crates/embeddings — ONNX embeddings, KNN, cosine similarity
  └── crates/ui-physics — Rapier3D world, spring solver, FPS mode
```

### State Management

**Frontend (Zustand):**
- 9 store slices: chat, notes, graph, settings, pipeline, research, ui, vault, system
- Event bus for cross-slice communication
- All backend data fetched via `invoke()`, never direct DB access

**Backend (AppState):**
```rust
pub struct AppState {
    pub db: Arc<Mutex<Database>>,
    pub physics: Arc<Mutex<PhysicsWorld>>,
    pub graph: Arc<Mutex<GraphStore>>,
    pub embeddings: Arc<Mutex<EmbeddingStore>>,
    pub physics_running: Arc<AtomicBool>,
    pub watcher: Arc<Mutex<Option<VaultWatcher>>>,
    pub cost_tracker: Arc<Mutex<CostTracker>>,
    pub enrichment_cancel: Arc<Mutex<Option<CancellationToken>>>,
    pub shutdown_token: CancellationToken,
    pub task_handles: Arc<Mutex<Vec<(&'static str, JoinHandle<()>)>>>,
}
```

**Lock ordering (prevents deadlock):** db → graph → embeddings → physics → watcher → cost_tracker

### IPC Pattern

```rust
// Backend: Tauri command
#[tauri::command]
#[specta::specta]
async fn create_page(state: State<'_, AppState>, title: String) -> Result<Page, AppError> {
    let db = state.db.lock().await;
    let page = Page::new(title);
    db.insert_page(&page)?;
    Ok(page)
}

// Frontend: invoke
const page = await invoke<Page>("create_page", { title: "My Note" });
```

### Streaming Pattern (for LLM/pipeline)

```rust
// Backend: emit events
app.emit("chat-stream", StreamChunk { text: "hello", done: false })?;
app.emit("pipeline-stage", StageEvent { stage: "SYNTHESIS", active: true })?;

// Frontend: listen
import { listen } from "@tauri-apps/api/event";
listen("chat-stream", (event) => {
    useChatStore.getState().appendStreamingText(event.payload.text);
});
```

**CRITICAL:** Never use per-item invoke() loops. Always batch:
```rust
// WRONG: 100 IPC round-trips
for note in notes { invoke("create_page", { title: note.title }) }

// RIGHT: 1 IPC round-trip
invoke("create_pages_batch", { pages: notes })
```

## Mac → Retro Translation Guide

| Mac Pattern | Retro Equivalent |
|-------------|-----------------|
| `@Observable class FooState` | Zustand store slice |
| `@Environment(FooState.self)` | `useFooStore()` hook |
| `SwiftData @Query` | `invoke("list_pages")` |
| `page.loadBody()` | `invoke("load_body", { id })` |
| `page.saveBody(text)` | `invoke("save_body", { id, content })` |
| `Task { @MainActor in }` | `useEffect` + `invoke()` |
| `AsyncStream<String>` | `listen("event-name", callback)` |
| `NotificationCenter.post` | Zustand event bus or Tauri events |
| `.sheet(isPresented:)` | Dialog component (Radix) |
| `.popover()` | Popover component (Radix) |
| `GraphState.engineHandle` (Rust FFI) | `AppState.physics` (Rapier3D) |
| `AppleIntelligenceService` | Foundry Local / Ollama via triage |
| `TriageService.shouldFallbackToAPI()` | `triage.rs` complexity check |
| `NoteWindowManager.open(pageId:)` | React Router navigate + store |
| `.glassEffect()` | Tailwind glassmorphism classes |

### Key Behavioral Differences
1. **No multi-window.** Mac opens notes in separate windows. Retro uses tabs/routes in one window.
2. **No Metal shaders.** Graph rendering uses Bevy + wgpu + WGSL instead of Metal.
3. **No Apple Intelligence.** On-device inference uses Foundry Local (Windows NPU) or Ollama.
4. **No SwiftData.** All persistence is rusqlite with explicit SQL. No auto-migration.
5. **Typed IPC boundary.** Every data type crossing the Tauri bridge must be serializable via serde + specta.

## Code Quality Rules

### Rust
- No `.unwrap()` in production code — use `?` or `.expect("context")`
- No copy-paste between modules — extract shared logic
- `cargo clippy -- -D warnings` must pass
- `cargo test` must pass before any commit
- `// SAFETY:` comment required on every `unsafe` block
- `with_capacity()` for all Vec allocations in hot paths
- Zero `clone()` in render loops — borrow or use indices

### TypeScript
- `tsc --noEmit` must pass (strict mode)
- No `any` types — use proper interfaces from bindings.ts
- No `console.log` outside debug-logger.ts
- Prefer `invoke()` over any other backend communication
- Use Zustand selectors to prevent unnecessary re-renders

### General
- DRY ruthlessly — if code repeats, extract it
- Names are documentation — comment only non-obvious WHY
- Error handling at boundaries only — trust internal code
- Async over sync everywhere
- Test-first: write failing test before the fix

## 3-Pass Pipeline (CRITICAL — this is the core product)

```
Pass 1 (Streaming, user-facing):
  → Parse <thinking> tags → deliberation events
  → Extract [CONCEPTS: ...] → concept list
  → Advance through 10 stages (TRIAGE → CALIBRATION)
  → Yield text deltas for live chat display

Pass 2 (Background, 180s timeout):
  → Epistemic Lens: 6,000-token analytical prose
  → Evidence tiers, Bayesian reasoning, effect sizes
  → Uses enrichmentSnapshot() for provider selection

Pass 3 (Background, 300s timeout):
  → Consolidated JSON:
    { laymanSummary, reflection,
      arbitration: { votes: [{framework, confidence, reasoning}] },
      truthAssessment: { overallLikelihood, factors } }
  → Full fallback on parse failure (signal-derived defaults)

INVARIANT: Pass 1 cancels on new query. Passes 2-3 continue in background.
Results delivered via Tauri events, not return values.
```

### 10 Pipeline Stages
TRIAGE → MEMORY → ROUTING → STATISTICAL → CAUSAL → META_ANALYSIS → BAYESIAN → SYNTHESIS → ADVERSARIAL → CALIBRATION

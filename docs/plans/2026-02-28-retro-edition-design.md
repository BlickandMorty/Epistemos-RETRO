# Epistemos Retro Edition — Architecture Design

> A Tauri 2.x + Bevy + Rapier3D + Vite knowledge graph app for Windows.
> Translates logic from Epistemos macOS (Swift/Rust) and UI from Brainiac 2.0
> (Next.js) into a unified native desktop application.

---

## Architecture Overview

```
Tauri 2.x Application
├── Native Window (TAO)
│   ├── Bevy + wgpu Render Surface (graph, physics, visual effects, UI glow)
│   │   └── Rapier3D physics (graph nodes + UI spring solver)
│   └── Transparent Webview Overlay (Vite + React: sidebar, chat, editor)
│       └── Rust WASM spring solver (button/panel physics, same engine params)
├── Rust Backend (shared across both surfaces)
│   ├── storage/     — rusqlite + file-based note bodies
│   ├── engine/      — LLM providers + 3-pass pipeline + triage
│   ├── graph/       — GraphStore + GraphBuilder + EntityExtractor
│   ├── sync/        — Vault watcher + BlockParser + BlockReconciler + Search
│   ├── embeddings/  — ort crate + DirectML (NPU/GPU)
│   └── ui-physics/  — Spring solver (dual-target: WASM + native)
└── Frontend (Vite + React 19 + React Router + Tailwind 4)
    ├── Brainiac 2.0 UI (ported from Next.js)
    ├── tauri-specta auto-generated bindings
    ├── Zustand 13-slice store (copied as-is)
    ├── Custom theme provider (6 themes)
    └── CodeMirror 6 editor (block refs, transclusion)
```

### Dual-Surface Rendering

The application window has two layers:

1. **Bevy render surface** (bottom): Opaque. Renders the knowledge graph with
   WGSL shaders, Rapier3D physics, and visual effects (glow, particles,
   animated backgrounds). This is the "retro" experience — GPU-driven, stylish.

2. **Transparent webview** (top): Renders the React UI — sidebar, chat panel,
   note editor, settings. Background is transparent, so Bevy's visuals show
   through. Text and interactive elements are crisp DOM rendering.

Interaction flow: React detects user interactions (clicks, hovers, drags) and
sends them to Rust via invoke(). Rust/Bevy handles the response (physics
simulation, data mutation, LLM calls) and pushes updates back via events.

## Crate Dependency Graph

```
                    src-tauri (application crate)
                   /    |    \       \        \
                  /     |     \       \        \
            engine   graph   sync  embeddings  ui-physics
               \      / \    /
                \    /   \  /
               storage

            graph-render (standalone Bevy app, reads graph store via Arc)
```

**Rule: arrows point downward. No crate depends on a crate above it.**
**graph-render** is a sibling, not a child — it reads `GraphStore` via
`Arc<RwLock<GraphStore>>` shared through `AppState`.

### crates/graph

In-memory graph store, structural builder, AI entity extraction.

```rust

impl GraphStore {

    pub fn connected(&self, node_id: &GraphNodeId, max_depth: usize) -> Vec<&GraphNode>;
    pub fn shortest_path(&self, from: &GraphNodeId, to: &GraphNodeId) -> Option<Vec<GraphNodeId>>;

```

**GraphBuilder** — deterministic structural graph from storage:
```rust
pub struct GraphBuilder;

impl GraphBuilder {
    pub fn build(storage: &Storage) -> Result<GraphSnapshot, GraphError>;
    pub fn persist(storage: &Storage, snapshot: &GraphSnapshot) -> Result<DiffStats, GraphError>;
}
```

Build order: Pages → Tags → Ideas → Blocks → ((blockId)) refs → Folders →
Folder hierarchy → Note-folder edges → Nested pages → Chats.

`persist()` is diff-based: compares by `"type-sourceId"` key, only applies
inserts/updates/deletes. Manual nodes (`is_manual: true`) are never touched.

**EntityExtractor** — AI-powered, batches 5 pages:

Annotates body text with `[block:UUID]` markers, asks LLM for JSON

(supports/contradicts/expands/questions/cites).

**Dependencies:** `storage` (crate), `engine` (crate, for LlmProvider trait),

**Vault import** is incremental: compares file modification dates vs `Page.updated_at`,
only processes new/modified/deleted files. Parses YAML front-matter, extracts title/tags.

**Dual-layer search:** FST (via `fst` crate) for sub-millisecond fuzzy label
matching. FTS5 (via rusqlite) for BM25-ranked full-text search with field weights
(title=5.0, body=1.0, tags=2.0). A single `search()` call runs both and merges
results.

**Dependencies:** `storage` (crate), `notify`, `fst`, `rusqlite` (bundled),
`serde`, `thiserror`

## Data Flows

### Chat Query Lifecycle

```
User types query in React
  → commands.submitQuery({ chatId, query, controls })     [tauri-specta]
  → Rust submit_query command:
    → storage.save_message(user_msg)                       [rusqlite]
    → broadcast::channel(256)
    → tokio::spawn pipeline:
      → engine::Pipeline::run(provider, query, chat_id, controls, tx)
        → QueryAnalyzer::analyze(query, context)           [pure Rust]
        → SignalGenerator::generate(analysis)              [pure Rust]
        → tx.send(Stage { triage })
        → provider.stream(request)                         [reqwest → cloud]
          → for chunk: tx.send(TextDelta { text })
        → tx.send(Complete { minimal_dual_message })
        → tokio::spawn enrichment:
          → Pass 2: generate_raw_analysis()                [reqwest, 180s]
          → Pass 3: generate_consolidated_enrichment()     [reqwest, 300s]
          → tx.send(Enriched { dual_message, truth })
    → Subscriber 1: while Ok(e) = rx.recv() → app_handle.emit(e)
    → Subscriber 2: on Complete → storage.save_message(assistant_msg)
                     on Enriched → storage.update_message(enrichment)
    → Returns Ok(()) immediately
  ← listen('pipeline://text-delta') → store.appendStreamingText()
  ← listen('pipeline://complete')   → store.completeProcessing()
```

### Note Save Lifecycle

```
User edits note → 5s debounce
  → commands.saveBody({ pageId, content })                 [tauri-specta]
  → Rust save_body command:
    → storage::note_body::write(page_id, content)          [atomic file write]
    → sync::BlockReconciler::reconcile(storage, page_id, content)
      → parse_blocks(content)                              [O(n) Rust]
      → Bipartite match by Jaccard ≥ 0.4
      → Stable BlockIds preserved, new blocks created
    → storage.update_page(page_id, { updated_at })
    → search_index.upsert(page, content)                   [FST + FTS5]
    → vault_tx.send(VaultEvent::PageSaved(page_id))
  → Subscriber: vault_watcher.export_page(page_id, vault_path)
```

### Graph Build Lifecycle

```
App launch / vault change / manual rebuild
  → commands.rebuildGraph()
  → Rust rebuild_graph command:
    → tokio::spawn_blocking:
      → graph::GraphBuilder::build(storage)
        → Pages → Note nodes (weight: word_count / 100)

        → Blocks → Block nodes + Contains edges

        → Folders → Folder nodes + hierarchy
        → Chats → Chat nodes

    → app_handle.emit("graph://updated", stats)
  ← listen('graph://updated') → UI refreshes graph view
```
